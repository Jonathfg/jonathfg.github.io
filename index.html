Justificación del Framework

¿Para qué sirve Angular?

Angular es un framework de desarrollo web mantenido por Google, orientado a la construcción de aplicaciones robustas, escalables y modernas del lado del cliente. Sirve para crear aplicaciones de una sola página (SPAs) con experiencia fluida, interfaces complejas y mantenibles, y optimizar la productividad del equipo.

¿Qué aplicaciones tiene?

Angular se utiliza en un amplio abanico de contextos: aplicaciones empresariales internas (ERP, CRM), paneles de administración, sitios de e-commerce interactivos, plataformas de educación en línea, apps móviles híbridas con Ionic, y soluciones escalables de misión crítica.

¿Sobre qué lenguajes se apoya?

Angular se basa principalmente en TypeScript, un superconjunto tipado de JavaScript, además de HTML y CSS para las vistas y el diseño. Esta combinación ofrece robustez, mantenibilidad y compatibilidad con el ecosistema web moderno.

¿Qué prerrequisitos necesita?

Se recomienda conocimientos básicos de HTML, CSS, JavaScript y estar dispuesto a aprender TypeScript. También es necesario tener Node.js y npm instalados, y entender la arquitectura típica de una aplicación web con componentes y servicios.

Instalación

¿Se precisa instalación?

Sí. Necesitas instalar Node.js (con npm) y el Angular CLI para crear, compilar y servir proyectos Angular fácilmente.

Pasos para la instalación

En Windows:

  1. Descargar la versión LTS de Node.js desde nodejs.org.
  2. Verificar con node -v y npm -v.
  3. Instalar Angular CLI: npm install -g @angular/cli.

En macOS/Linux:

  1. Instalar Node.js (por ejemplo, usando nvm).
  2. Verificar con node -v y npm -v.
  3. Instalar Angular CLI: sudo npm install -g @angular/cli.

Versiones necesarias

Se recomienda usar la versión LTS de Node.js y la última versión estable de Angular para garantizar compatibilidad y soporte a largo plazo.

Rutas a tener en cuenta y variables de entorno

Al crear un proyecto (ng new) se genera una estructura con src/app (código principal), src/assets (estáticos), src/environments (config. por entorno). Angular no requiere variables de entorno específicas por defecto, pero es posible definir endpoints o configuraciones personalizadas en environments/.

¿Necesita base de datos?

Angular es frontend, no se conecta directamente a la base de datos. Utiliza APIs REST/GraphQL. Si se necesita persistencia, se conecta a un backend o servicios externos. No requiere base de datos directa del lado del cliente.

Primeros Pasos

Preparación del Entorno de Desarrollo

Asegúrate de tener Node.js y Angular CLI instalados. Es recomendable usar un editor como VSCode con extensiones para Angular y TypeScript. Familiarízate con la línea de comandos y ng para generar componentes y servicios.

Creación de un Proyecto Angular

Para crear un nuevo proyecto:

ng new mi-primera-app
cd mi-primera-app
ng serve --open

Esto iniciará el servidor en http://localhost:4200 y mostrará la página inicial de Angular.

Estructura de un Proyecto Angular

La estructura típica incluye app.module.ts (módulo principal), app.component.ts (componente raíz), app.component.html (vista raíz), app.component.css (estilos), y otros componentes y servicios en src/app. Los recursos estáticos van en src/assets.

Primeros Pasos en la Codificación (Hola Mundo)

El "Hola Mundo" en Angular es la página inicial generada por CLI. Edita app.component.html y reemplaza su contenido con:

<h1>¡Hola Mundo desde Angular!</h1>

Guarda y el navegador mostrará tu mensaje. Esta es la primera aproximación y confirmación de que el entorno funciona.

Utilización del Framework

¿Cómo empezar a crear una aplicación base?

Tras el "Hola Mundo", empieza a crear componentes con ng generate component, define rutas con ng generate module app-routing --flat --module=app, añade servicios para lógica de negocio y consume APIs. La aplicación base surge al organizar componentes en módulos y enlazarlos con el enrutamiento.

¿Cómo visualizar los resultados de la ejecución?

Con ng serve tienes recarga en vivo en http://localhost:4200. Para producción, usa ng build --configuration production y despliega los archivos estáticos resultantes en un hosting. Durante el desarrollo, cada cambio guardado se refleja automáticamente en el navegador.

Funcionamiento Interno

¿Actúa sobre páginas, ficheros, secciones…?

Angular trabaja con componentes, cada uno con su HTML, CSS y TS. Estos componentes se agrupan en módulos. El enrutador simula múltiples páginas sin recargas. Cada componente representa una sección de la interfaz, y la aplicación se considera una SPA.

¿Cómo se editan/modifican los ficheros?

Edita el HTML, TS o CSS de cada componente según necesites. Usa data binding ({{variable}}), directivas (*ngIf, *ngFor), y servicios inyectados para gestionar la lógica y los datos. Cada cambio se ve reflejado inmediatamente en el navegador durante el desarrollo.

¿Cómo se referencian los ficheros desde otras aplicaciones?

Si integras Angular en una aplicación mayor, normalmente sirves el build de Angular (carpeta dist) como estáticos. Si otra app requiere consumir el frontend Angular, se integra a través de rutas proxy o iframes. Las dependencias se gestionan con npm y el Angular CLI.

Conclusiones

Hemos respondido las preguntas clave sobre Angular: su utilidad, las aplicaciones, en qué lenguajes se basa, qué se necesita para empezar, cómo se instala, cómo crear un proyecto, ejecutar el "Hola Mundo", comprender su arquitectura de componentes y módulos, y cómo visualizar resultados. Con estas bases, un principiante puede iniciar con confianza el desarrollo de aplicaciones complejas y escalables en Angular.

Enrutamiento

Angular proporciona un sistema de enrutamiento interno para crear una SPA con múltiples vistas. Configura rutas en app-routing.module.ts asignando componentes a rutas (e.g. { path: 'home', component: HomeComponent }). El enrutador evita recargas completas y facilita la navegación fluida.

Consumo de APIs

Utiliza el servicio HttpClient para hacer peticiones HTTP a servicios REST. Inyecta HttpClient en tus servicios y usa métodos como http.get(), http.post() para interactuar con el backend. Esto permite mostrar datos dinámicos en la vista.

Módulos y Lazy Loading

Organiza tu aplicación en módulos para separar funcionalidades. El lazy loading carga módulos bajo demanda, mejorando el rendimiento inicial. Configura rutas con loadChildren y Angular cargará el módulo cuando se acceda a la ruta correspondiente.

Directivas y Pipes

Las directivas manipulan el DOM dinámicamente (*ngFor, *ngIf) o añaden comportamiento. Los pipes transforman datos en la vista (| date, | uppercase). Creando directivas y pipes personalizados aumentas la reutilización y limpieza del código.

Pruebas (Testing)

Angular incluye soporte para pruebas unitarias con Jasmine y Karma. Ejecuta ng test para correr pruebas. Crea pruebas unitarias para componentes, servicios y pipes. También puedes hacer pruebas end-to-end (E2E) con Protractor para validar el funcionamiento en navegadores reales.

Despliegue

Tras usar ng build --configuration production, obtienes archivos estáticos en dist/. Sube estos archivos a un hosting estático (e.g. GitHub Pages, Netlify, Firebase Hosting) o a tu propio servidor. Angular no requiere un backend especial para servir el frontend compilado.

Seguridad y Autenticación

Protege tu aplicación contra ataques XSS usando data binding seguro. Implementa autenticación y autorización con JWT, OAuth u otros. Intercepta peticiones HTTP con HttpInterceptors para añadir tokens. Usa guardas de ruta (CanActivate) para restringir acceso a secciones protegidas.

SEO y Metadatos

Las SPAs son desafiantes para el SEO. Usa Angular Universal para renderizado del lado del servidor (SSR) y mejorar el SEO. Añade metadatos, Open Graph y microdatos para optimizar la visibilidad en buscadores y redes sociales.

Internacionalización

Traduce tu aplicación a varios idiomas con el módulo @angular/localize. Extrae texto, genera archivos de traducción (XLF) y compila la app para distintos locales. Esto permite llegar a audiencias globales.

Rendimiento y Optimización

Optimiza el rendimiento usando ChangeDetectionStrategy.OnPush, lazy loading de módulos e imágenes, preload de rutas, y herramientas como Lighthouse para medir. Minimiza el uso de zonas pesadas, evita cálculos costosos en plantillas, y aprovéchate del Ivy Renderer para builds más rápidos.

Accesibilidad

Asegura que tu app sea usable por todos. Añade atributos ARIA, usa contrastes adecuados, soporte para teclado y lectores de pantalla. Angular no hace accesibilidad por sí solo, pero tu código debe cumplir buenas prácticas (etiquetas semánticas, roles, etc.).

Gestión de Estado

Aplicaciones complejas requieren manejar estados compartidos entre componentes. Usa RxJS y NgRx para un estado global predecible, patrones Redux, acciones, efectos y reducers. Esto facilita el debugging y la escalabilidad.

Animaciones

Angular incorpora el módulo de animaciones basado en Web Animations API. Crea transiciones suaves entre vistas, animaciones en la entrada/salida de elementos con @angular/animations. Esto mejora la experiencia de usuario.

Herramientas del Ecosistema

Aprovecha Angular CLI para generar código, crear builds optimizados, ejecutar pruebas, y filtrar proyectos. Usa extensiones como Angular Language Service en VSCode para autocompletado, Angular Console para interfaz gráfica, y bibliotecas de componentes (Material, PrimeNG) para acelerar el desarrollo.

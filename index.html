Empezar a hacer tu primera aplicación
1

Justificación del Framework

¿Para qué sirve Angular?
Angular es un framework para el desarrollo de aplicaciones web modernas de una sola página (SPA). Permite crear interfaces dinámicas, componentes reutilizables, manejar estados complejos y ofrecer experiencias de usuario fluidas sin recargar la página. Esto lo hace ideal para crear entornos más parecidos a aplicaciones de escritorio que a sitios web tradicionales.

Con Angular, la actualización de datos en la interfaz es inmediata. Cuando un usuario interactúa con el contenido, los componentes pueden cambiar su estado y reflejar los cambios sin necesidad de una recarga completa. Esta filosofía incrementa la eficiencia y la satisfacción del usuario.

¿Qué aplicaciones tiene?
Angular se aplica en múltiples contextos:

  • Paneles internos de administración y dashboards que requieren datos actualizados en tiempo real.
  • Aplicaciones de e-commerce con carritos dinámicos, filtros instantáneos y actualizaciones de stock sin recarga.
  • Plataformas de aprendizaje en línea (LMS) con módulos interactivos, evaluaciones y contenido que cambia según el progreso del alumno.
  • Aplicaciones móviles híbridas (con Ionic) que aprovechan la base de Angular para desplegar en móviles y web.
  • Sistemas empresariales complejos (ERP, CRM) que requieren escalabilidad, mantenibilidad y modularidad.
En definitiva, cualquier aplicación web que necesite experiencias ricas del lado del cliente se beneficia de Angular.

¿Sobre qué lenguajes se apoya?
Angular se basa principalmente en TypeScript (un superconjunto tipado de JavaScript). Esto facilita el mantenimiento, ya que el tipado estático ayuda a detectar errores temprano, ofrece mejor autocompletado y documentación interna. Para la vista, utiliza HTML y CSS, los lenguajes estándar de la web. Esto significa que no tienes que aprender un lenguaje de plantillas complejo; usas lo que ya conoces, enriquecido con directivas y bindings proporcionados por Angular.

¿Qué prerrequisitos necesita?
Para empezar con Angular es ideal tener:

  • HTML y CSS: Conocimientos básicos para construir la interfaz.
  • JavaScript: Familiaridad con la programación del lado del cliente.
  • TypeScript: Disposición a aprender su tipado, clases e interfaces. Es similar a JavaScript, pero con tipos, lo que reduce errores.
  • Node.js y npm: Instalados para usar el CLI de Angular, gestionar dependencias y scripts.
Con esto, podrás entrar con confianza en el mundo de Angular y aprovechar su potencial, construyendo aplicaciones más limpias, mantenibles y escalables.

2

Instalación

¿Se precisa instalación?
Sí, necesitas Node.js, npm y el Angular CLI. El CLI simplifica la creación y mantenimiento de proyectos Angular: con comandos simples generas componentes, servicios y módulos, y levantas un servidor de desarrollo que se actualiza al instante con cada cambio.

Pasos para la instalación:
Primero instala Angular CLI: 

npm install -g @angular/cli
ng version
Con esto instalas el CLI globalmente y verificas la versión. Asegúrate de tener Node.js (versión LTS). El CLI también facilita la ejecución de pruebas, la generación de builds optimizados y el despliegue.

Versiones necesarias:
Usa la versión LTS de Node.js para mayor estabilidad. Angular lanza versiones regularmente, pero empezar con la última versión estable es lo más aconsejable. Tendrás acceso a la documentación más reciente y la comunidad más activa.

Rutas a tener en cuenta:
Al crear un proyecto con ng new:

  • src/app: Código principal (componentes, servicios, módulos). Aquí ocurre la magia de tu aplicación.
  • src/assets: Archivos estáticos (imágenes, fuentes). Tus recursos para estilos y contenido estático.
  • src/environments: Configuraciones para entornos (dev, prod). Aquí puedes colocar endpoints distintos según el entorno.
Esta organización facilita mantener orden y escalabilidad a medida que tu app crece.

Variables de entorno:
No obligatorias, pero útiles para configurar endpoints (por ejemplo, una API distinta para desarrollo y otra para producción) o claves secretas. Aunque Angular no las maneja por defecto al estilo Node.js, puedes usar distintos archivos environment.ts o técnicas con Angular CLI para establecer ajustes.

¿Necesita base de datos?
No directamente. Angular es frontend, se encarga de la vista y la lógica del cliente. La persistencia de datos ocurre en un backend (API REST, GraphQL u otros servicios). Angular se integra con esas APIs a través de HttpClient u otras librerías. Puedes simular datos localmente con JSON o librerías mock, pero la lógica persistente está fuera de Angular.

3

Primeros pasos

“Hola Mundo”:
Crea un proyecto y ejecútalo en el servidor de desarrollo: 

ng new mi-primera-app
cd mi-primera-app
ng serve --open
Esto abre el navegador en http://localhost:4200. Edita src/app/app.component.html: 
<h1>Hola Mundo desde Angular!</h1>
Verás el mensaje reflejado inmediatamente al guardar. Este primer paso te confirma que el entorno está funcionando y que el CLI hizo su trabajo.

Explora la estructura del proyecto:

  • app.module.ts: Declara componentes, módulos y servicios que tu app necesita para arrancar.
  • app.component.ts: Define el componente principal (AppComponent), su lógica y propiedades.
  • app.component.html: La plantilla del componente principal, donde muestras el “Hola Mundo”.
Entender esta base te preparará para construir apps más complejas: pronto crearás múltiples componentes y rutas para dividir tu aplicación en secciones más manejables.

4

Utilización

¿Cómo crear una aplicación base?
Para expandir tu aplicación, genera componentes con el CLI: 

ng generate component mi-componente
Esto crea los archivos TS, HTML, CSS y el test del componente automáticamente. Configurar rutas es igual de sencillo: 
ng generate module app-routing --flat --module=app
Define rutas en app-routing.module.ts, asociando rutas a componentes. Un usuario puede navegar entre secciones sin recargas, mejorando la experiencia.

¿Cómo visualizar resultados?
En desarrollo: 

ng serve --open
Al guardar cambios, el navegador se actualiza. Para producción: 
ng build --configuration production
Esto genera archivos optimizados en dist/. Estos archivos puedes subirlos a un hosting estático (GitHub Pages, Netlify, etc.) y tu app estará lista para que cualquier usuario acceda. Así separas la etapa de desarrollo (con recarga en vivo) de la etapa de despliegue (archivos optimizados).

La utilización del framework sigue este patrón: CLI para tareas comunes, HTML+CSS+TS para la lógica y vistas, y un entorno flexible para crecer en complejidad a medida que lo necesites.

5

Funcionamiento Interno

¿Actúa sobre páginas, ficheros, secciones…?
Angular funciona con componentes. Cada componente tiene:

  • Un archivo TS con la clase del componente (propiedades, métodos, hooks del ciclo de vida).
  • Una plantilla HTML donde se definen vistas dinámicas.
  • Un archivo CSS (opcional) para estilos encapsulados.
Estos componentes se agrupan en módulos que organizan la app. El enrutador asigna cada ruta a un componente, simulando múltiples páginas internas sin recarga completa.

¿Cómo se editan/modifican los ficheros?
Ajusta la lógica en el TS (por ejemplo, agregando una propiedad title), luego muestra esa propiedad en el HTML con {{ title }}. Si necesitas mostrar una lista, usa *ngFor, y si requieres mostrar algo condicional, usa *ngIf. Los estilos en CSS se aplican solo a ese componente, evitando conflictos con otros.

¿Cómo se referencian los ficheros desde otras aplicaciones?
Tras compilar, Angular produce HTML, JS y CSS estáticos. Otra aplicación puede consumir tu app servida en una URL específica, integrarla en un iframe o enlazarla en el menú principal. La salida final es puro web standard, sin dependencias de servidores especiales. Esto facilita la integración en entornos más complejos.

Conclusiones

Has aprendido:

  • Justificación: Por qué Angular es valioso, en qué casos brilla (SPAs complejas, UIs ricas, escalabilidad).
  • Instalación: Preparar el entorno con Node.js, CLI y entender la estructura clara de los proyectos Angular.
  • Primeros pasos: Crear un proyecto, mostrar “Hola Mundo” y examinar la arquitectura básica.
  • Utilización: Generar componentes, configurar rutas, ejecutar en modo desarrollo, compilar para producción y desplegar sin complicaciones.
  • Funcionamiento: Entender el modelo de componentes, módulos y enrutamiento interno, así como la edición de cada parte para construir UIs dinámicas y mantenibles.
Con estas bases, estás listo para profundizar en temas avanzados: consumir APIs externas, escribir pruebas unitarias y E2E, optimizar rendimiento, aplicar internacionalización, mejorar accesibilidad, agregar animaciones e integrar librerías de terceros. Angular es una potente herramienta en tu arsenal de desarrollador web moderno, y ahora tienes la base para aprovecharla al máximo.

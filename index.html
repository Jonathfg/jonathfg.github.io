Justificación del Framework

¿Para qué sirve Angular?

Angular es un framework de desarrollo web mantenido por Google, orientado a la construcción de aplicaciones robustas, escalables y modernas del lado del cliente. Sirve para:

  • Crear aplicaciones de una sola página (SPAs) con una experiencia de usuario fluida.
  • Desarrollar interfaces complejas, mantenibles y organizadas.
  • Optimizar el tiempo de desarrollo gracias a su arquitectura modular y herramientas integradas.

¿Qué aplicaciones tiene?

Angular se utiliza en una amplia gama de soluciones:

  • Sistemas internos de gestión empresarial, paneles de administración y dashboards.
  • Aplicaciones de e-commerce interactivas.
  • Aplicaciones de educación en línea y plataformas de e-learning.
  • Aplicaciones móviles híbridas con Ionic.
  • Aplicaciones escalables de misión crítica, gracias a su arquitectura sólida.

¿Sobre qué lenguajes se apoya?

Angular se sustenta sobre:

  • TypeScript: Lenguaje clave para la lógica y la programación, un superconjunto tipado de JavaScript.
  • HTML: Para las plantillas de los componentes y estructura de la interfaz.
  • CSS/SCSS: Para el estilo y la presentación visual de la aplicación.

¿Qué prerrequisitos necesita?

Para iniciar con Angular se recomienda:

  • Conocimientos básicos de HTML, CSS y JavaScript.
  • Familiaridad con TypeScript, o disposición para aprenderlo.
  • Tener instalado Node.js y npm.
  • Entender el concepto de componentes, servicios, enrutamiento y arquitectura de aplicaciones web.

Instalación

¿Se precisa instalación?

Sí. Es necesario instalar Node.js (que incluye npm) y el Angular CLI (Interface de Línea de Comandos) para crear, compilar y servir proyectos Angular de forma sencilla.

Pasos para la instalación

En Windows:

  1. Descargar e instalar la versión LTS de Node.js desde nodejs.org.
  2. Abrir el "Símbolo del sistema" o "PowerShell" y verificar la instalación: 
    node -v
npm -v
  3. Instalar Angular CLI globalmente: 
    npm install -g @angular/cli

En macOS/Linux:

  1. Instalar Node.js (usando el paquete oficial o un gestor como nvm): 
    node -v
npm -v
  2. Instalar Angular CLI: 
    sudo npm install -g @angular/cli

Versiones necesarias

Se recomienda usar la versión LTS de Node.js (por ejemplo, Node 18 LTS) y la última versión estable de Angular para garantizar compatibilidad y soporte.

Rutas a tener en cuenta

Al crear un proyecto con Angular CLI (ng new), se genera una estructura base:

  • src/app: Código principal de la aplicación (componentes, servicios, módulos).
  • src/assets: Archivos estáticos (imágenes, fuentes).
  • src/environments: Configuraciones para diferentes entornos (dev, prod).

Variables de entorno

Angular no requiere variables de entorno obligatorias. Sin embargo, puedes configurar endpoints de APIs u otras variables mediante los archivos en src/environments y definir distintas configuraciones para desarrollo, producción, etc.

¿Necesita base de datos?

No directamente. Angular es un framework frontend y no se conecta a la base de datos por sí mismo. En su lugar, consume servicios o APIs REST/GraphQL. Puedes usar servicios mock, APIs de terceros o crear tu propio backend separado.

Primeros Pasos

Creando un “Hola Mundo”

Con Angular CLI instalado, crear un nuevo proyecto es sencillo:

ng new mi-primera-app
cd mi-primera-app
ng serve --open

Esto abrirá el navegador en http://localhost:4200 mostrando la página inicial de Angular, que podemos considerar nuestro "Hola Mundo".

Utilización del Framework

¿Cómo empezar a crear una aplicación base?

La estructura creada por ng new ya incluye un componente principal llamado AppComponent. En src/app/app.component.html puedes editar el texto para personalizar el mensaje de bienvenida:

<h1>¡Hola Mundo desde Angular!</h1>

¿Cómo visualizar los resultados?

Con el comando ng serve --open la aplicación se recarga automáticamente al guardar cambios. Para visualizar en producción, se usa:

ng build --configuration production

Esto genera la carpeta dist lista para desplegar en un hosting estático.

Funcionamiento Interno

¿Actúa sobre páginas, ficheros, secciones…?

Angular organiza el frontend en componentes. Cada componente tiene:

  • Un archivo TypeScript (.ts) con la lógica.
  • Un archivo HTML (.html) con la plantilla de la vista.
  • Un archivo CSS/SCSS (.css) para el estilo.

Estos componentes se agrupan en módulos, y la aplicación se considera una SPA, con enrutamiento interno para simular múltiples páginas sin recargas completas.

¿Cómo se editan/modifican los ficheros que actúan en el framework?

Para editar un componente, simplemente ajusta el HTML, el TS o el CSS correspondientes. El framework usa data binding con {{ variable }}, directivas (*ngIf, *ngFor), e inyección de dependencias para servicios.

¿Cómo se referencian los ficheros creados desde otras aplicaciones?

Si necesitas integrar Angular con una aplicación mayor, usualmente el build de Angular se coloca en un servidor y se integra a través de rutas reversas o proxies. Las dependencias se gestionan con npm y el Angular CLI. Los componentes se importan en sus módulos, y los servicios se inyectan en los componentes que los requieran. Si hay recursos estáticos, se colocan en assets/ y se acceden vía rutas relativas.

Conclusiones

A lo largo de este tutorial se han presentado las bases para entender qué es Angular, para qué sirve, cómo se instala, cómo iniciar un proyecto y cómo estructurar su código. Con esto se sientan las bases para profundizar y comenzar a desarrollar aplicaciones completas.

Se han logrado los objetivos propuestos: comprender la razón de ser del framework, sus casos de uso, el entorno necesario, los primeros pasos, la estructura del proyecto y el funcionamiento interno. Estas nociones iniciales permitirán al lector abordar proyectos más complejos y sacar provecho de las numerosas capacidades de Angular.

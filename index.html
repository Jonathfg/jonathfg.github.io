Empezar a hacer tu primera aplicación
1

Justificación del Framework

¿Para qué sirve? Angular sirve para desarrollar aplicaciones web modernas, dinámicas y escalables, ofreciendo una arquitectura sólida y componentes reutilizables.

¿Qué aplicaciones tiene? Desde paneles internos de administración, aplicaciones de una sola página (SPAs), e-commerce, hasta aplicaciones móviles híbridas con Ionic.

¿Sobre qué lenguajes se apoya? Principalmente en TypeScript, además de HTML y CSS para la vista y el estilo.

¿Qué prerrequisitos necesita? Conocimientos básicos de HTML, CSS, JavaScript, disposición para aprender TypeScript, y tener instalado Node.js y npm.

2

Instalación

¿Se precisa instalación? Sí, necesitas Node.js, npm y Angular CLI.

Pasos para la instalación: npm install -g @angular/cli, verifica con ng version.

Versiones necesarias: Usa la versión LTS de Node.js y la última versión estable de Angular.

Rutas a tener en cuenta: src/app para componentes, src/assets para estáticos, src/environments para configuraciones.

Variables de entorno: No obligatorias, pero útiles para endpoints u otras configuraciones.

¿Necesita base de datos? No directamente, Angular consume APIs externas.

3

Primeros pasos

“Hola Mundo”: Crea un proyecto con ng new mi-primera-app, ejecuta ng serve --open y edita app.component.html para mostrar “Hola Mundo desde Angular!”

4

Utilización del Framework

¿Cómo crear una aplicación base? Usa componentes generados con la CLI, define rutas, servicios y vistas.

¿Cómo visualizar resultados? Con ng serve en http://localhost:4200 con recarga en vivo. Para producción, ng build --configuration production.

5

Funcionamiento Interno

¿Actúa sobre páginas, ficheros, secciones…? Angular actúa sobre componentes organizados en módulos. El enrutador simula múltiples páginas sin recarga.

¿Cómo se editan los ficheros? Cada componente tiene su archivo TS, HTML y CSS. Usa data binding, directivas, pipes y servicios.

¿Cómo se referencian ficheros desde otras apps? Normalmente despliegas la app Angular como estáticos; otras apps pueden consumirla vía proxies o iframes.

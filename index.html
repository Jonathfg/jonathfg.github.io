Justificación del Framework

¿Para qué sirve Angular?
Angular es un framework para el desarrollo de aplicaciones web modernas de una sola página (SPA). Permite crear interfaces dinámicas, componentes reutilizables, manejar estados complejos y ofrecer experiencias de usuario fluidas sin recargar la página. Esto lo hace ideal para crear entornos más parecidos a aplicaciones de escritorio que a sitios web tradicionales.

Con Angular, la actualización de datos en la interfaz es inmediata. Cuando un usuario interactúa con el contenido, los componentes pueden cambiar su estado y reflejar los cambios sin necesidad de una recarga completa. Esta filosofía incrementa la eficiencia y la satisfacción del usuario.

¿Qué aplicaciones tiene?
Angular se aplica en múltiples contextos:

  • Paneles internos de administración y dashboards.
  • Aplicaciones de e-commerce con carritos dinámicos, filtros instantáneos y actualizaciones de stock sin recarga.
  • Plataformas de aprendizaje en línea (LMS) con módulos interactivos y evaluaciones dinámicas.
  • Aplicaciones móviles híbridas (con Ionic) que aprovechan Angular para desplegar en móviles y web.
  • Sistemas empresariales complejos (ERP, CRM) que requieren escalabilidad y modularidad.
En definitiva, cualquier aplicación web que necesite experiencias ricas del lado del cliente se beneficia de Angular.

¿Sobre qué lenguajes se apoya?
Angular se basa en TypeScript (un superconjunto tipado de JavaScript). Esto facilita el mantenimiento, ya que el tipado estático reduce errores. Para la vista, usa HTML y CSS, lenguajes estándar. No necesitas aprender templating complejo: Angular potencia lo que ya conoces con directivas y data binding.

¿Qué prerrequisitos necesita?
Para empezar con Angular es ideal:

  • HTML y CSS básicos.
  • JavaScript para entender la lógica del lado del cliente.
  • TypeScript: aprender su tipado, clases e interfaces brinda un entorno más robusto.
  • Node.js y npm: para usar el CLI, gestionar dependencias y scripts.
Con esto, entrarás con confianza en Angular, construyendo apps más limpias y escalables.

Instalación

Un poco de historia:
Angular fue creado por Miško Hevery y el equipo de Google. La primera versión (AngularJS) fue lanzada en 2010. Con el tiempo, el framework evolucionó hacia Angular (sin JS) a partir de la versión 2 (2016), reescribiéndolo en TypeScript. Hoy en día, Angular es mantenido por Google y la comunidad, con actualizaciones regulares que mantienen el framework moderno y eficiente.

¿Se precisa instalación?
Sí, necesitas Node.js, npm y el Angular CLI. El CLI simplifica la creación de proyectos, la generación de componentes y el despliegue.

Instalar en distintos sistemas operativos:

  • Windows: Descarga Node.js desde su sitio oficial e instálalo. Luego, en "PowerShell" o "CMD": 
    npm install -g @angular/cli
ng version
  • macOS: Usa Homebrew para instalar Node.js: 
    brew install node
npm install -g @angular/cli
    Comprueba con ng version.
  • Linux (Ubuntu/Debian): Instala Node.js desde los repositorios oficiales o via nvm, luego: 
    sudo npm install -g @angular/cli
ng version
Así tendrás el CLI listo en cualquier SO.

Versiones necesarias:
Usa Node.js LTS y la última versión estable de Angular. Esto te asegura compatibilidad y acceso a las últimas funcionalidades.

Rutas importantes:
Al crear un proyecto con ng new:

  • src/app: Código principal (componentes, servicios, módulos).
  • src/assets: Archivos estáticos (imágenes, fuentes).
  • src/environments: Configuraciones para diferentes entornos (dev, prod).
Esta organización facilita la escalabilidad.

Variables de entorno:
No obligatorias, pero útiles para endpoints o claves. Usa environment.ts para definir variables según el entorno.

¿Necesita base de datos?
No. Angular es frontend puro. La lógica de datos se maneja en el backend. Angular consume APIs REST/GraphQL. Puedes simular datos localmente, pero la persistencia real ocurre en el servidor.

Primeros pasos

“Hola Mundo” en Angular:
Crea un proyecto y ejecútalo: 

ng new mi-primera-app
cd mi-primera-app
ng serve --open
Se abrirá http://localhost:4200. Edita src/app/app.component.html: 
<h1>Hola Mundo desde Angular!</h1>
El mensaje aparece al instante.

Crear una aplicación desde cero:
Un principiante puede empezar con el "Hola Mundo". Luego, agrega componentes con: 

ng generate component mi-componente
Ajusta app.component.html para mostrar tu componente. Usa ng serve para ver cambios al instante. Conforme aprendas, agregarás servicios para manejar lógica de datos, módulos para organizar tu app, y rutas para dividir la app en secciones. Con paciencia y práctica, tu pequeña app “Hola Mundo” se convertirá en algo mucho más completo.

Explora la estructura del proyecto:

  • app.module.ts: Declara y agrupa componentes, servicios, etc.
  • app.component.ts: Componente principal.
  • app.component.html: Plantilla del componente principal.
Esta base sienta las pautas para apps más complejas.

4

Utilización

¿Cómo crear una aplicación base?
Usa el CLI para generar componentes y módulos. Por ejemplo: 

ng generate component header
ng generate component footer
Ahora tienes un header y footer listos. Integra rutas: 
ng generate module app-routing --flat --module=app
Define rutas en app-routing.module.ts para navegar sin recargas.

Visualizar resultados:
Durante el desarrollo: 

ng serve --open
Para producción: 
ng build --configuration production
Esto crea archivos optimizados en dist/. Sube esos archivos a un hosting estático. Así separas el desarrollo del despliegue final.

La potencia del CLI y el patrón de componentes y módulos te permite crecer de una simple app a una compleja, manteniendo claridad y modularidad.

5

Funcionamiento Interno

¿Actúa sobre páginas, ficheros, secciones…?
Angular usa componentes (TS, HTML, CSS). Se agrupan en módulos. El enrutador asigna rutas a componentes, simulando páginas internas sin recarga. Cada componente es como un bloque Lego que arma tu aplicación.

¿Cómo se editan/modifican ficheros?
Ajusta el TS para la lógica (propiedades, métodos), el HTML para la vista dinámica (con {{ variable }}, *ngIf, *ngFor) y el CSS para estilos locales. Esto aísla la lógica, la vista y los estilos, mejorando el mantenimiento.

¿Cómo se integran con otras apps?
Tras compilar, tu app Angular es HTML, JS y CSS estáticos. Otra app puede integrarla vía iframe, enlace, o servidor. La salida final es puro web standard, sin dependencias extra, facilitando la integración en entornos complejos.

Historia y Evolución de Angular

Angular nació como AngularJS en 2010, creado por Miško Hevery y el equipo de Google. Fue revolucionario por introducir el concepto de data binding bidireccional y un enfoque declarativo para construir interfaces web. Con el tiempo, la comunidad y Google vieron la necesidad de un framework más modular, escalable y basado en TypeScript, lo que llevó al lanzamiento de Angular 2 en 2016. Desde entonces, Angular ha evolucionado con versiones regulares (cada 6 meses aproximadamente), proporcionando nuevas características, mejoras de rendimiento y mejores prácticas, sin romper la compatibilidad de forma radical.

Hoy en día, Angular es un framework maduro, con un ecosistema rico (CLI, librerías de terceros, Angular Material, NgRx para estado, etc.) y una comunidad activa que contribuye en foros, GitHub, conferencias y workshops. Este soporte continuo garantiza que Angular se mantenga relevante y robusto, preparado para los desafíos del desarrollo web moderno.

Ejemplo Completo: Creando una Aplicación desde Cero

Ahora que tienes las bases, crearemos una pequeña aplicación “Lista de Tareas” para consolidar el aprendizaje. Esta app permitirá agregar, listar y marcar tareas como completadas.

1. Crear el Proyecto:

ng new lista-tareas
cd lista-tareas
ng serve --open
Abre http://localhost:4200. Verás la app base de Angular.

2. Generar un Componente Tareas:

ng generate component tareas
Esto crea tareas.component.ts, tareas.component.html, tareas.component.css y un test.

3. Definir la Lógica de Tareas:
En tareas.component.ts, agrega un array de tareas (objetos con texto y estado), y métodos para agregar y togglear tareas.

4. Plantilla HTML:
En tareas.component.html, crea un formulario para agregar una tarea, y una lista con *ngFor que muestre las tareas y un botón para marcarla completada. Usa (submit) y (click) para manejar eventos.

5. Integrar el Componente:
En app.component.html, añade la etiqueta <app-tareas></app-tareas>. Guarda y revisa en el navegador. Ahora tienes una pequeña app funcional, con estado, eventos y vista dinámica.

Este ejemplo consolida lo aprendido: CLI para generar componentes, bindings para mostrar datos, eventos para cambiar estado y vistas reactivas. A medida que crezcas en Angular, podrás añadir servicios para separar la lógica de datos, rutas para múltiples páginas internas, y librerías de terceros para gráficos, autenticación, etc. ¡El cielo es el límite!

BEGINNER COURSE

Angular

Aprende a dominar Angular desde sus fundamentos. Este tutorial, con un estilo retro, te guiará paso a paso para que comprendas por qué Angular es tan poderoso, cómo instalarlo, crear tu primera aplicación, entender su funcionamiento interno y sacar conclusiones útiles para avanzar con solidez.

Empezar a hacer tu primera aplicación
1

Justificación del Framework

¿Para qué sirve Angular?
Angular es un framework para el desarrollo de aplicaciones web modernas de una sola página (SPA). Permite crear interfaces dinámicas, componentes reutilizables, manejar estados complejos y ofrecer experiencias de usuario fluidas sin necesidad de recargar completamente la página. Esto hace que el usuario interactúe con una aplicación más parecida a un entorno de escritorio que a un sitio web tradicional.

¿Qué aplicaciones tiene?
Angular se aplica en una amplia gama de proyectos:

  • Paneles internos de administración y dashboards.
  • Aplicaciones de e-commerce con funcionalidades interactivas avanzadas.
  • Plataformas de aprendizaje en línea con módulos dinámicos.
  • Aplicaciones móviles híbridas a través de Ionic.
  • Sistemas empresariales complejos que requieren escalabilidad y mantenibilidad.
En definitiva, cualquier aplicación web que requiera experiencias ricas del lado del cliente se beneficia de Angular.

¿Sobre qué lenguajes se apoya?
Angular se basa principalmente en TypeScript, un superconjunto tipado de JavaScript. Esto le da mayor robustez y facilidad de mantenimiento a largo plazo. Para la vista, usa HTML y CSS, de modo que no necesitarás un lenguaje nuevo para la presentación, sólo para la lógica y la arquitectura. TypeScript te ofrece un entorno más predecible, con autocompletado, verificación de tipos y una integración perfecta con el ecosistema JavaScript.

¿Qué prerrequisitos necesita?
Para empezar con Angular, es ideal tener:

  • Conocimientos básicos de HTML y CSS para la presentación de la interfaz.
  • Familiaridad con JavaScript, ya que TypeScript se construye sobre él.
  • Disposición para aprender TypeScript, entender su tipado, interfaces y clases.
  • Tener instalado Node.js y npm para administrar paquetes y ejecutar el CLI de Angular.
Con esto, podrás entrar sin problemas en el mundo de Angular y aprovechar su potencial.

2

Instalación

¿Se precisa instalación?
Sí, necesitarás Node.js, npm y el Angular CLI (Command Line Interface). El CLI facilita la creación de proyectos, la generación de componentes, la ejecución de la aplicación en desarrollo y el empaquetado para producción.

Pasos para la instalación:
Primero, asegúrate de tener Node.js instalado. Luego, en la terminal, ejecuta: 

npm install -g @angular/cli
Esto instalará el CLI de Angular globalmente. Comprueba con: 
ng version
para ver la versión instalada.

Versiones necesarias:
Usa la versión LTS de Node.js para mayor estabilidad. Angular lanza versiones periódicas, así que la última versión estable es recomendable para empezar, contando con su documentación y soporte actualizados.

Rutas a tener en cuenta:
Cuando creas un proyecto con ng new, se genera una estructura estándar:

  • src/app: Código principal (componentes, servicios, módulos).
  • src/assets: Archivos estáticos (imágenes, fuentes, etc.).
  • src/environments: Archivos de configuración para distintos entornos (desarrollo, producción).
Esta organización facilita el mantenimiento y escalado.

Variables de entorno:
No son obligatorias, pero pueden ser útiles para almacenar endpoints de APIs, claves o configuraciones según el entorno (dev, prod). Angular no las requiere por defecto, pero su estructura y la comunidad ofrecen formas de implementarlas fácilmente.

¿Necesita base de datos?
No directamente. Angular es frontend puro. Para datos persistentes, se conecta a APIs REST o GraphQL de un backend. Puedes simular datos con herramientas locales, pero la lógica de persistencia corre en el servidor, no en Angular.

3

Primeros pasos

“Hola Mundo”:
Una vez instalado el CLI, crea un proyecto: 

ng new mi-primera-app
cd mi-primera-app
ng serve --open
Esto abrirá el navegador en http://localhost:4200 mostrando la página inicial de Angular. Edita src/app/app.component.html y reemplaza su contenido por: 
<h1>Hola Mundo desde Angular!</h1>
Al guardar, verás el mensaje en el navegador. Esta es tu primera confirmación de que todo funciona.

Además del “Hola Mundo”, dedica unos minutos a explorar la estructura del proyecto. Observa el app.module.ts, donde se declaran componentes, y app.component.ts con la clase del componente principal. Comprender esta arquitectura te ayudará a sentar bases sólidas.

4

Utilización del Framework

¿Cómo crear una aplicación base?
Utiliza el CLI para generar componentes: 

ng generate component mi-componente
Esto creará un directorio con TS, HTML, CSS y pruebas del componente. Integra estos componentes en tu app.module.ts y ajusta las vistas. Para navegación, genera un módulo de enrutamiento: 
ng generate module app-routing --flat --module=app
Allí defines rutas y componentes asociados.

¿Cómo visualizar resultados?
Durante el desarrollo, ejecuta: 

ng serve --open
Cada vez que guardes un cambio, el navegador se actualizará automáticamente. Para un entorno productivo: 
ng build --configuration production
Esto genera en dist/ los archivos listos para ser desplegados en un servidor estático. Así podrás publicar tu aplicación sin necesidad de un backend especial, sólo necesitas un hosting de archivos.

5

Funcionamiento Interno

¿Actúa sobre páginas, ficheros, secciones…?
Angular trabaja con componentes, que son pequeñas piezas de interfaz. Cada componente tiene:

  • Un archivo TypeScript para la lógica (propiedades, métodos, ciclo de vida).
  • Un archivo HTML para la plantilla (vista).
  • Un archivo CSS opcional para estilos encapsulados.
Estos componentes se combinan en módulos, y el enrutador permite navegar entre ellos sin recargar la página, simulando múltiples secciones o “páginas internas”.

¿Cómo se editan/modifican los ficheros?
Simplemente abre los archivos TS, HTML o CSS del componente que quieras cambiar. Ajusta la lógica en el TS, la vista en el HTML y el estilo en el CSS. Usa data binding (p. ej., {{variable}}) y directivas como *ngIf y *ngFor para mostrar datos dinámicos o listas. Servicios inyectados ofrecen datos y lógica compartida.

¿Cómo se referencian los ficheros desde otras aplicaciones?
Normalmente, despliegas la aplicación Angular como un conjunto de archivos estáticos optimizados. Otras aplicaciones pueden interactuar con ella de diferentes maneras, por ejemplo, sirviéndola en una ruta específica del servidor. Angular no impone restricciones; su build final es puro HTML, CSS y JavaScript que cualquier servidor puede servir. Si otra app quiere integrar la app Angular, puede hacerlo mediante iframes, redirecciones o integración en la capa del servidor.

6

Conclusiones

En este punto, has comprendido:

  • La justificación de Angular: Por qué existe, qué problemas resuelve y en qué casos es útil.
  • La instalación: Cómo preparar tu entorno para empezar a desarrollar con Angular.
  • Los primeros pasos: Crear un proyecto, levantar el servidor de desarrollo y mostrar un “Hola Mundo”.
  • La utilización del framework: Cómo generar componentes, configurar rutas, visualizar cambios en tiempo real y preparar la app para producción.
  • El funcionamiento interno: Entender la arquitectura basada en componentes, módulos y enrutamiento, así como la forma de editar y referenciar ficheros.
Con estas bases, estás listo para adentrarte en temas más profundos. Angular es extenso: desde enrutamiento avanzado, consumo de APIs, optimización, pruebas y despliegue, hasta internacionalización, gestión de estado, animaciones y más. Lo aprendido aquí te da el trampolín para seguir creciendo y crear aplicaciones sólidas y mantenibles.

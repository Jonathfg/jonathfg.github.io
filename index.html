BEGINNER COURSE

Angular

Aprende a dominar Angular desde sus fundamentos. Este tutorial, con un estilo retro, te guiará paso a paso para que comprendas por qué Angular es tan poderoso, cómo instalarlo, crear tu primera aplicación, entender su funcionamiento interno y sacar conclusiones útiles para avanzar con solidez.

Empezar a hacer tu primera aplicación
1

Justificación del Framework

¿Para qué sirve Angular?
Angular es un framework para el desarrollo de aplicaciones web modernas de una sola página (SPA). Permite crear interfaces dinámicas, componentes reutilizables, manejar estados complejos y ofrecer experiencias de usuario fluidas sin necesidad de recargar completamente la página. Esto hace que el usuario interactúe con una aplicación más parecida a un entorno de escritorio que a un sitio web tradicional.

¿Qué aplicaciones tiene?
Angular se aplica en una amplia gama de proyectos:

  • Paneles internos de administración y dashboards.
  • Aplicaciones de e-commerce con funcionalidades interactivas avanzadas.
  • Plataformas de aprendizaje en línea con módulos dinámicos.
  • Aplicaciones móviles híbridas a través de Ionic.
  • Sistemas empresariales complejos que requieren escalabilidad y mantenibilidad.
En definitiva, cualquier aplicación web que requiera experiencias ricas del lado del cliente se beneficia de Angular.

¿Sobre qué lenguajes se apoya?
Angular se basa principalmente en TypeScript, un superconjunto tipado de JavaScript. Esto le da mayor robustez y facilidad de mantenimiento a largo plazo. Para la vista, usa HTML y CSS, así no necesitas un lenguaje nuevo para la presentación, sólo para la lógica y la arquitectura. TypeScript te ofrece un entorno más predecible, con autocompletado, verificación de tipos y una integración perfecta con el ecosistema JavaScript.

¿Qué prerrequisitos necesita?
Para empezar con Angular, es ideal tener:

  • Conocimientos básicos de HTML y CSS para la presentación de la interfaz.
  • Familiaridad con JavaScript, ya que TypeScript se construye sobre él.
  • Disposición para aprender TypeScript (tipado, interfaces y clases).
  • Tener instalado Node.js y npm para usar el CLI de Angular.
Con esto, podrás entrar sin problemas en el mundo de Angular y aprovechar su potencial.

2

Instalación

¿Se precisa instalación?
Sí, necesitarás Node.js, npm y el Angular CLI. El CLI facilita la creación de proyectos, generación de componentes, ejecución en desarrollo y empaquetado para producción.

Pasos para la instalación:
Primero, asegúrate de tener Node.js. Luego, en la terminal: 

npm install -g @angular/cli
ng version
Con esto instalas el CLI globalmente y verificas la versión.

Versiones necesarias:
Usa la versión LTS de Node.js. Para Angular, la última versión estable es lo más recomendable, así cuentas con la documentación y soporte actualizados.

Rutas a tener en cuenta:
Al crear un proyecto con ng new:

  • src/app: Código principal (componentes, servicios, módulos).
  • src/assets: Estáticos (imágenes, fuentes).
  • src/environments: Configuraciones para distintos entornos.
Esta organización facilita el mantenimiento y escalado.

Variables de entorno:
No obligatorias, pero puedes usarlas para endpoints de APIs o configuraciones sensibles. Angular no las requiere por defecto, pero la comunidad y la estructura del proyecto facilitan su implementación.

¿Necesita base de datos?
No directamente. Angular es frontend puro. Para datos persistentes, se conecta a un backend o servicios externos via APIs REST o GraphQL.

3

Primeros pasos

“Hola Mundo”:
Crea un proyecto: 

ng new mi-primera-app
cd mi-primera-app
ng serve --open
Esto abre el navegador en http://localhost:4200. Edita src/app/app.component.html: 
<h1>Hola Mundo desde Angular!</h1>
Verás el mensaje reflejado inmediatamente.

Explora la estructura del proyecto: revisa app.module.ts (donde se declaran componentes y módulos), y app.component.ts (la clase del componente principal). Entender esta base te ayudará a construir aplicaciones más complejas con seguridad.

4

Utilización del Framework

¿Cómo crear una aplicación base?
Usa el CLI para generar componentes: 

ng generate component mi-componente
Esto crea TS, HTML, CSS y pruebas del componente. Configura enrutamiento: 
ng generate module app-routing --flat --module=app
Define rutas en el app-routing.module.ts y asocia componentes a ellas.

¿Cómo visualizar resultados?
En desarrollo: 

ng serve --open
Al guardar cambios, el navegador se actualiza. Para producción: 
ng build --configuration production
Genera archivos estáticos en dist/. Despliega estos en un servidor estático y listo, sin necesidad de un backend especial, salvo el que consuma la app si requieres datos.

5

Funcionamiento Interno

¿Actúa sobre páginas, ficheros, secciones…?
Angular está basado en componentes, cada uno con su propia lógica (TS), vista (HTML) y estilos (CSS). Los componentes se organizan en módulos. El enrutador simula múltiples páginas sin recargar: el usuario navega entre rutas definidas sin una recarga completa, logrando una experiencia más ágil.

¿Cómo se editan/modifican los ficheros?
Abre el archivo TS para la lógica (variables, métodos, ciclo de vida), el HTML para la plantilla (vistas dinámicas con data binding, directivas *ngIf, *ngFor) y el CSS para los estilos. Ajusta lo necesario y guarda: el servidor de desarrollo reflejará los cambios al instante.

¿Cómo se referencian los ficheros desde otras aplicaciones?
Una vez compilada, la app Angular es HTML, CSS y JS estático. Puedes integrarla en otro sitio vía iframes, o servirla en una ruta especial del servidor. Otras apps pueden interactuar con ella a través de su URL, sin complicaciones. Angular no impone trabas; su salida final es un conjunto de archivos estáticos independientes.

6

Conclusiones

Has aprendido:

  • Justificación: Por qué Angular y en qué contextos brilla.
  • Instalación: Cómo preparar tu entorno (Node.js, Angular CLI) y entender la estructura del proyecto.
  • Primeros pasos: Crear un proyecto, mostrar “Hola Mundo” y familiarizarte con su arquitectura.
  • Utilización: Generar componentes, configurar rutas, ver resultados en tiempo real y compilar para producción.
  • Funcionamiento interno: Comprender su arquitectura basada en componentes, módulos, rutas y cómo editar cada parte.
Con estas bases, estás listo para explorar aspectos avanzados: enrutamiento detallado, consumo de APIs, pruebas, despliegue, rendimiento, accesibilidad, internacionalización y más. Lo fundamental ya lo tienes: el entendimiento del porqué, el cómo y el qué de Angular, marcando el camino para construir aplicaciones poderosas y mantenibles.
